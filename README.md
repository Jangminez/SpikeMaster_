# SpikeMaster (스파이크 마스터)
> **인터페이스 기반 다형성 설계와 비동기 코루틴 상태 머신으로 구현한 모바일 원터치 배구 게임**

구체적인 클래스 간의 결합도를 낮추고 모바일 환경 최적화를 달성하기 위해 추상화 아키텍처 및 런타임 캐싱을 도입하여 개발한 원터치 배구 게임입니다.

---

## 개발 환경 및 기술 스택
* Engine: Unity 6.3 LTS
* Language: C#
* Target Platform: Mobile (Android)
* Backend & Third-Party: Microsoft PlayFab, Google AdMob

---

## 시스템 아키텍처 및 핵심 설계

### 1. 인터페이스 기반 다형성 플레이어 구조 (IPlayer & BasePlayer)
* 역할 분담 및 결합도 제어: 공격수(PlayerSpiker), 세터(PlayerSetter), 리베로(PlayerLibero) 등 고유의 메커니즘을 가진 포지션들을 단일 스크립트 분기문 대신 다형성(Polymorphism)으로 설계했습니다.
* 확장성 극대화 (OCP): 상위 매니저인 PlaySceneManager나 PlayerHandler는 구체적인 캐릭터 타입을 모른 채 IPlayer 인터페이스 하나로 모든 캐릭터의 액션을 동일하게 제어할 수 있어 시스템 간 느슨한 결합(Decoupling)을 실현했습니다.

### 2. 중재자 패턴 기반 코루틴 상태 머신 (RallyFlowManager)
* 안정적인 랠리 시퀀스 제어: 서브, 리시브, 토스, 스파이크, 판정으로 이어지는 순차적 흐름을 Update문 플래그 연산 없이, 중앙 집중식 코루틴(RallyMainCO)과 WaitUntil 비동기 제어를 통해 직관적인 유한 상태 머신(FSM) 형태로 통제합니다.
* 단일 책임 원칙(SRP) 준수: RallyFlowManager는 상위 흐름만 중재하며 개별 연산은 하위 계층(PlayerHandler 에이전트 제어, EnemySpawner 랜덤 스폰 연산, RallyInputHandler 거리 기반 판정)으로 책임을 완전히 독립시켰습니다.

### 3. 글로벌 시스템의 선형 비동기 파이프라인 (GameManager)
* 초기화 순서 보장 (레이스 컨디션 방지): 글로벌 인프라 매니저들을 IBaseManager 인터페이스로 규격화하고, GameManager에서 코루틴 시퀀스를 통해 이전 매니저의 초기화 완료(IsInitialized)를 완전히 보장받은 후 다음 매니저를 깨우는 안전한 선형 초기화 구조를 구축했습니다.

## 주요 성능 최적화 (Optimization)

### 1. 애니메이션 런타임 오버헤드 최적화 (PlayerAnimHandler)
* 문자열 파싱 차단: 플레이어 캐릭터가 빈번히 애니메이션 상태를 전환할 때 발생하는 내부 문자열 연산 비용을 줄이기 위해, PlayerAnimHashes 정적(Static) 클래스 내에 Animator.StringToHash를 활용해 고유 해시값으로 사전 캐싱했습니다.
* 가비지 컬렉터(GC) 부하 최소화: 애니메이션 캡슐화와 해시 참조를 통해 런타임 힙 메모리 할당을 억제하여 모바일 환경의 프레임 드랍(스파이크 현상)을 방지했습니다.

### 2. 가변적 네트워크 지연 은닉 및 데이터 보안
* **AdMob 비동기 캐싱 (Preloading)**: 광고 로딩 시 발생하는 네트워크 지연으로 인한 UX 저하를 방지하고자 게임 시작 시점에 광고를 미리 백그라운드에서 비동기 로드하여 캐싱해 두는 파이프라인을 구축했습니다.
* **PlayFab 클라우드 데이터 동기화**: 로컬 저장 데이터 변조 위험을 배제하기 위해 유저의 데이터 및 재화를 PlayFab 클라우드 플랫폼과 실시간 동기화하여 데이터 무결성을 확보했습니다.

## 핵심 게임 메커니즘
* 거리 기반 다단계 터치 판정: 공과 플레이어의 실시간 거리를 연산하여 Perfect, Good, Bad, Miss 판정을 내리고 월드 좌표계에 실시간 이펙트를 출력합니다.
* 스파이크 타점 기반 동적 각도 연산: 스파이크 모드 진입 시 공의 어느 위치(타점)를 터치하냐에 따라 스파이크의 궤적과 공격 각도가 실시간으로 변화하도록 설계하여, 플레이어가 직접 공격 방향을 정교하게 컨트롤하는 손맛과 전략성을 강화했습니다.
* 실시간 동적 밸런싱 시스템: 플레이어가 터치 타이밍을 완벽하게 맞추거나 연속 콤보를 달성할수록 공의 이동 속도 배율(_dynamicSpeedMultiplier)과 득점 가중치가 유기적으로 상승하여 플레이의 박진감과 몰입도를 조절합니다.
